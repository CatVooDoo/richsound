Статус проекта Musykallity / Richsound

Актуально на текущий момент.

Реализовано

1. Базовая архитектура приложения
- Чистый PHP-проект с разделением на Router, Controllers, Models, Views.
- Подключение к MySQL через PDO.
- Маршрутизация через `src/routes/web.php`.

2. Аутентификация и роли
- Регистрация пользователя с ролями `listener` и `author`.
- Вход и выход из аккаунта.
- Сессии для авторизации.
- Роль `admin` добавлена в схему БД.
- Через обычную регистрацию роль `admin` получить нельзя.
- Админ может существовать только если роль выставлена напрямую в БД или через админку.

3. Главная страница
- Главная больше не статическая: на ней выводятся реальные треки из таблицы `tracks`.
- Есть блок свежих треков.
- Есть блок популярных треков.
- Есть featured-трек в верхнем hero-блоке.
- Если треков нет, показываются пустые состояния.

4. Плеер
- На главной работает общий плеер на основе HTMLAudioElement.
- Есть запуск трека по клику на карточку.
- Есть play/pause.
- Есть previous/next.
- Есть перемотка по progress bar.
- Есть управление громкостью.
- Есть обновление названия, исполнителя, длительности и обложки в плеере.
- Есть отправка события прослушивания на сервер.

5. Статистика прослушиваний
- Есть таблица `listens`.
- Подготовлен SQL для хранения статистики прослушиваний:
  - `track_id`
  - `user_id`
  - `listened_seconds`
  - `is_completed`
  - `listened_at`
- Добавлены индексы для аналитики.
- Добавлен SQL-триггер, который увеличивает `tracks.plays_count` после вставки в `listens`.

6. Админка
- Есть отдельная админ-панель `/admin`.
- Доступ только для пользователей с ролью `admin`.
- После логина админ отправляется в админку.
- В админке есть:
  - сводная статистика;
  - график прослушиваний;
  - график регистраций;
  - топ треков;
  - управление пользователями;
  - управление альбомами;
  - управление треками.
- Есть создание, редактирование и удаление пользователей.
- Есть создание, редактирование и удаление альбомов.
- Есть создание, редактирование и удаление треков.
- Есть защита от удаления последнего администратора.

7. Локализация интерфейса
- Основная используемая верстка переведена на русский.
- Исправлены пользовательские тексты на главной, в логине и регистрации.


Реализовано частично

1. Музыкальный каталог
- Треки уже выводятся на главной и участвуют в плеере.
- Но отдельных полноценных страниц каталога, поиска и фильтрации пока нет.

2. Статистика
- Базовая запись прослушиваний и агрегаты уже есть.
- Но нет глубокой аналитики по авторам, жанрам, retention, дослушиваниям и периодам.

3. Работа с файлами
- В БД предусмотрены пути к трекам и обложкам.
- Но полноценная загрузка MP3 и изображений через интерфейс автора пока не завершена.

4. Пользовательский опыт плеера
- Основная логика работает.
- Но нет persistent-плеера между разными страницами приложения.
- Нет очереди воспроизведения вне главной.
- Нет shuffle/repeat.
- Нет сохранения текущего состояния плеера в localStorage.


Не реализовано

1. Кабинет автора
- Нет законченного интерфейса автора для загрузки треков.
- Нет законченного интерфейса управления альбомами автора.
- Нет готового публичного профиля автора.
- Нет полноценной авторской аналитики по своим релизам.

2. Слушательские функции
- Нет полноценно работающих лайков.
- Нет избранного.
- Нет подписок на авторов через интерфейс.
- Нет создания и управления личными плейлистами.
- Нет добавления/удаления треков в плейлисты через интерфейс.

3. Контент-страницы
- Нет полноценных страниц:
  - трека;
  - альбома;
  - профиля автора;
  - поиска.

4. Стриминг и backend player flow
- Нет отдельного backend-стриминга аудио через PHP с range-headers.
- Сейчас плеер использует прямой путь к аудиофайлу из `file_path`.

5. Полноценная поисковая система
- Нет поиска по трекам, альбомам и авторам.
- Нет сортировок и фильтров в UI.

6. Полная бизнес-логика платформы
- Нет завершённого listener-flow.
- Нет завершённого author-flow.
- Нет завершённого production-ready admin-flow с разбиением на отдельные страницы и более тонкими правами.


Что важно проверить вручную

1. В БД должны быть реальные записи в `tracks`.
2. У треков должен быть корректный `file_path`, доступный из браузера.
3. Для существующей БД должны быть применены изменения из `src/sql.sql`.
4. Для админки у нужного пользователя должна быть роль `admin`.


Рекомендуемый следующий этап

1. Доделать загрузку треков и обложек через интерфейс.
2. Сделать страницы трека, альбома и автора.
3. Сделать поиск.
4. Доделать listener-функции: лайки, избранное, плейлисты, подписки.
5. Улучшить плеер: persistent state, очередь, repeat/shuffle.
